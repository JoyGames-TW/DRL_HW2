# Cliff Walking: Q-learning vs SARSA

## 一、作業目的
本專案實作並比較兩種經典強化學習演算法：**Q-learning**（離策略）與 **SARSA**（同策略）。透過經典的「Cliff Walking (懸崖尋路)」環境，使用相同的參數設定，分析其學習行為、收斂特性以及最終策略差異，並建立一個可部署至 Vercel 的 Web 視覺化應用程式。

## 二、環境與問題設定
* **環境**：4 × 12 網格世界 (Gridworld)
* **起點 (Start)**：左下角 (3, 0)
* **終點 (Goal)**：右下角 (3, 11)
* **懸崖 (Cliff)**：起點與終點之間的底部區域 (3, 1) 到 (3, 10)。若步入懸崖會獲得 -100 獎勵並回到起點。
* **獎勵機制**：每移動一步 -1 獎勵。
* **動作空間**：上 (0)、右 (1)、下 (2)、左 (3)
* **策略**：$\epsilon$-greedy
* **超參數設定**：
  * 學習率 ($\alpha$) = 0.5
  * 探索率 ($\epsilon$) = 0.1
  * 折扣因子 ($\gamma$) = 0.9
  * 訓練回合數 (Episodes) = 500

## 三、演算法實作過程
本專案使用 **Python** 實作後端 API (基於 Flask 架構)，並建置前段 Vanilla HTML/JS/CSS 進行互動與圖表展示。
1. **環境建置**：自行實作 `CliffWalkingEnv`，提供 `reset()` 與 `step()` 介面。
2. **代理人訓練**：分別實作 `q_learning` 與 `sarsa` 的學習迴圈。為降低單次執行的隨機性，數據為進行 20 次獨立訓練 (Runs) 取平均的結果。
3. **網頁介面與部署**：設計深色玻璃擬態 (Glassmorphism) 風格的現代化 UI，準備部署至 Vercel Serverless 環境中。前端負責向 Python API 請求計算結果，並利用 Chart.js 與 DOM 渲染學習曲線及最終策略路徑圖。

## 四、結果分析與比較

### 1. 學習表現 (累積獎勵曲線與收斂速度)
觀察訓練過程的累積獎勵曲線：
* **SARSA**：在經過初期的探索後，大約在數十回合內就會迅速收斂。最終收斂的平均每回合獎勵約在 **-17** 左右（因其走較安全的遠路，需花費較多步數）。
* **Q-learning**：雖然同樣會收斂，但其平均每回合獎勵僅能維持在約 **-40** 到 **-60** 之間震盪。這是因為 Q-learning 學到的是沿著懸崖邊的最短路徑（最佳獎勵為 -13），但在 $\epsilon=0.1$ 的探索下，每一步都有可能隨機走向懸崖，承受巨大的 -100 懲罰，導致平均表現不佳。

### 2. 策略行為 (最終路徑)
從前端渲染出來的最終路徑 (Policy Grid) 來看：
* **Q-learning (傾向冒險)**：計算出的最終策略直接沿著懸崖的正上方邊緣前進（最短路徑）。
* **SARSA (傾向保守)**：計算出的最終策略會引導代理人先往上方移動，遠離懸崖邊緣，走一條距離較長但絕對安全的路線。

### 3. 穩定性分析
* **較為穩定：SARSA**。SARSA 在訓練過程中的單回合獎勵波動極小，因為它及早學會了避開高風險區域。
* **波動劇烈：Q-learning**。Q-learning 由於採用最短的高風險路徑，即使在「收斂」後，依然會因為 $\epsilon$-greedy 的隨機探索而不斷掉入懸崖，導致過程極不穩定。

## 五、理論比較與討論
* **Q-learning 為離策略 (Off-policy)**：其 Q-value 更新公式中使用了 `max Q(S', a)`，亦即它總是假設「下一步會採取理論上最完美的行動」，而忽略了代理人實際上正在使用包含隨機性的 $\epsilon$-greedy 策略。因此，它會毫不猶豫地選擇緊貼懸崖的最佳路徑，但在實際運行時卻會因為探索而不斷摔下懸崖。
* **SARSA 為同策略 (On-policy)**：其 Q-value 更新公式中使用了**實際選定的下一步行動** `Q(S', A')`。如果下一步行動是隨機選出來且可能導致掉入懸崖的，SARSA 就會把這個「潛在風險」直接算進當前狀態的價值中，藉此學會遠離懸崖邊緣。這是一種更具**風險意識 (Risk-averse)** 的學習方式。

## 六、結論總結
1. **收斂較快**：**SARSA**，能更快找到穩定的安全策略。
2. **較穩定**：**SARSA**，在訓練過程中避免了頻繁的高額懲罰。
3. **何時該選擇 Q-learning**：如果環境中「探索 (Exploration) 的代價很低」，且我們的最終目標是在訓練完成後**完全關閉探索**（設 $\epsilon = 0$）來執行任務，那麼 Q-learning 能夠找出理論上的絕對最佳解。
4. **何時該選擇 SARSA**：如果在訓練或部署過程中**犯錯的代價極高**（如：真實世界機器人導航、自駕車），或是系統必須在持續保持探索 ($\epsilon > 0$) 的情況下運作，應選擇 SARSA 以獲得較安全、穩定的實務表現。
